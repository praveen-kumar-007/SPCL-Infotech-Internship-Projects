Internship: Build a Car Dealership Website

This short 10-day internship project helps you learn practical front-end & basic back-end skills: responsive layouts, search & filters, gallery handling, and a contact flow. Submit a GitHub repo and a live deployment (Netlify/Vercel/GitHub Pages).

Overview

Design and develop a responsive car dealership website that showcases listings with images, pricing, and filters. Use semantic HTML, modular CSS and clear JavaScript structure. Host your site (Netlify / Vercel / GitHub Pages) and link the repo in your submission.

Deliverables

  • Responsive homepage with featured carousel and navigation
  • Listings page with search, filters and consistent card layout
  • Car detail page with gallery and thumbnails
  • Contact form that persists or sends messages
  • README with run/deploy instructions and a short screencast or demo link

Timeline & Milestones

  1. Day 1–2: Project setup, layout & hero, implement basic listings
  2. Day 3–5: Filters, search and pagination; responsive testing
  3. Day 6–8: Detail page, image gallery, quick view modal
  4. Day 9: Polishing, accessibility, README
  5. Day 10: Deploy and submit repo/demo link

Evaluation

We evaluate on Design (30%), Functionality (30%), Code Quality (25%), and Presentation (15%). See the checklist below to ensure all important parts are covered.

  • Responsive layout, consistent cards, image fallbacks
  • Search and filters work correctly and are debounced
  • Accessible components (aria labels, focus states)
  • Clean repo history and deploy link

How to Apply

  1. Fork this project or create your own repo following the brief.
  2. Complete the features and push commits with clear messages.
  3. Deploy your site and ensure the contact form works.
  4. Submit the GitHub repo URL and live demo link in the form below.
